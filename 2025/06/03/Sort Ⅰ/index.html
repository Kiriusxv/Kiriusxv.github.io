​ 几个排序算法

🔸 插入排序（Insertion Sort）

基本思路： 将当前元素插入到前面已经排好序的部分中。

情况 时间复杂度
最好情况 O(n)
平均情况 O(n²)
最坏情况 O(n²)
空间复杂度 O(1)
稳定性 ✅ 稳定

优化：

  • 当数组基本有序时，插入排序性能非常好。
  • 可以用二分查找优化插入位置（但写法稍复杂）。

🔸 希尔排序（Shell Sort）

基本思路： 将数组按一定间隔分组做插入排序，逐渐减小间隔到1。

情况 时间复杂度
最好情况 O(n log n)
平均情况 O(n^1.3) ~ O(n²)
最坏情况 O(n²)
空间复杂度 O(1)
稳定性 ❌ 不稳定

优化：

  • 不同的“增量序列”会影响性能，如：Shell、Hibbard、Sedgewick 等。
  • 一般建议使用 gap = gap / 2 的方式即可满足普通需求。

🔸 冒泡排序（Bubble Sort）

基本思路： 每轮从前往后，两两比较，把最大的“冒”到最后。

情况 时间复杂度
最好情况 O(n)
平均情况 O(n²)
最坏情况 O(n²)
空间复杂度 O(1)
稳定性 ✅ 稳定

优化：

  • 加一个 flag，如果某一轮没有发生交换，说明已经有序，可以提前结束。

🔸 快速排序（Quick Sort）

基本思路： 选一个“基准值”，把比它小的放左边，大的放右边，然后递归。

情况 时间复杂度
最好情况 O(n log n)
平均情况 O(n log n)
最坏情况 O(n²)
空间复杂度 O(log n)
稳定性 ❌ 不稳定

优化：

  • 随机化选取 pivot（避免最坏情况如已排序数组）
  • 小数组可用插排代替递归（如 n < 10）
  • 使用“三数取中法”选 pivot 更稳健
